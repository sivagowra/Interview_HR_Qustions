AWS Interview Question

How can two EC2 instances communicate using private IP addresses without a VPC peering connection?

Answer

  • I have two EC2 instances launched.
  • The first EC2 instance has a public IP enabled.
  • The second EC2 instance has the public IP disabled.
  • I copied the private IP address of the second EC2 instance.
  • I connected to the first EC2 instance using a browser-based terminal.
  • From the first EC2 instance, I pinged the private IP of the second EC2 instance.
  • The ping works successfully without a VPC peering connection.

üß† What is User Data?

User Data is a startup script that runs automatically when an EC2 instance starts. It is mainly used to install software, configure services, and prepare the server without manual login.

User Data runs as the root user and is executed during the boot process using a service called cloud-init.

You add User Data while launching an EC2 instance or by editing it after stopping the instance. The script is written in bash shell format.

Example use cases include installing Docker, Git, Nginx, Java, or starting applications automatically.

üê≥ Docker Storage Types Explained

Docker containers are temporary by default. When a container is removed, its data is also removed. Docker storage helps us save data outside the container so that it is not lost. Docker provides three main types of storage to handle different use cases.

1Ô∏è‚É£ Docker Volumes

Docker volumes are the most commonly used and recommended storage type in Docker. A volume is created and managed directly by Docker and stored in a special Docker directory. When a container writes data to a volume, the data is stored safely outside the container. Even if the container stops or is deleted, the data inside the volume is not lost. First, a volume is created using a Docker command or automatically during container creation. Next, the volume is attached to a directory inside the container. The container then reads and writes data to that directory normally. Docker volumes work the same on any machine, which makes them reliable. They are easy to back up and restore when needed. Docker volumes are best suited for production applications and databases.

2Ô∏è‚É£ Bind Mounts

Bind mounts directly connect a folder on the host machine to a folder inside the container. In this method, Docker does not manage the storage location. The developer chooses an existing directory on the host system. When the container starts, that directory is mounted inside the container. Any changes made inside the container are immediately reflected on the host. This is very useful during development because code changes appear instantly. Bind mounts make it easy to edit files using local tools. However, they depend heavily on the host file system structure. This makes them less portable between different machines. Bind mounts are mainly used in development environments.

3Ô∏è‚É£ tmpfs Mounts

tmpfs mounts store data directly in the system memory instead of on disk. When a container writes data to a tmpfs mount, it is stored in RAM. This data is very fast to access because it does not involve disk operations. tmpfs storage is temporary and exists only while the container is running. Once the container stops or restarts, all data in tmpfs is removed. This type of storage is useful for sensitive information. It is often used for temporary cache data or secret files. Since data is never written to disk, security is improved. tmpfs mounts reduce disk usage on the host system. They are best used when data does not need to be saved.
