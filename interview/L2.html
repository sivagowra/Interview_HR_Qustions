L2 Technical Support Engineer (Linux) – Interview Q&A

1) Explain your general troubleshooting process

First, I understand the issue clearly from the user.

I reproduce the issue if possible.

I check application, system, and service logs.

I isolate the affected component.

I form hypotheses based on symptoms.

I apply the fix carefully.

I validate the solution.

Finally, I document the RCA and resolution.

2) How do you check disk usage?

df -h shows overall disk usage.

du -sh /path/* shows folder size.

find command finds large files.

Useful for cleanup and troubleshooting.

df -h
du -sh /var/*
find / -type f -size +500M

3) Commands for system performance

top shows real-time CPU usage.

htop gives interactive view.

free -m checks memory.

uptime shows load average.

top
htop
free -m
uptime

4) Explain rwx permissions

r = read (4)

w = write (2)

x = execute (1)

Permissions apply to user, group, others.

chmod changes permissions.

chown changes ownership.

5) How do you restart Linux services?

Using systemctl on systemd systems.

systemctl restart service_name
systemctl status service_name

6) How do you handle kernel issues?

Check kernel logs using dmesg.

Review /var/log/messages.

Check disk health using smartctl.

Run memory tests if required.

Identify hardware or driver issues.

7) TCP vs UDP

TCP is reliable and connection-oriented.

It ensures data delivery.

UDP is faster but unreliable.

No handshake or retransmission.

TCP used for HTTP, SSH.

UDP used for DNS, streaming.

8) How do you diagnose DNS issues?

Ping checks connectivity.

nslookup resolves DNS.

dig gives detailed DNS info.

Check /etc/resolv.conf.

ping google.com
nslookup google.com
dig google.com
cat /etc/resolv.conf

9) How do you troubleshoot latency?

Ping checks latency.

Traceroute finds network hops.

Check bandwidth usage.

Analyze server load.

Check network congestion.

10) How do you check DB connectivity?

Test port connectivity.

Login using DB client.

Check firewall rules.

telnet db_host 3306
mysql -h db_host -u user -p

12) Incident vs Problem vs Change

Incident: Service interruption.

Problem: Root cause of incidents.

Change: Planned modification.

13) Handling multiple high-severity issues

Prioritize based on SLA.

Check business impact.

Assign parallel tasks.

Communicate regularly.

14) How do you prepare RCA?

Describe the issue.

Explain business impact.

Identify root cause.

Explain fix.

List preventive actions.

15) Handling frustrated customers

Listen carefully.

Show empathy.

Provide clear ETA.

Give regular updates.

Remain calm and professional.

CPU and memory usage.

Application logs.

Database performance.

17) What is Grafana?

Grafana is a monitoring and visualization tool.

It displays metrics and logs.

Supports real-time dashboards.

Used for performance monitoring.

Helps in troubleshooting.

18) Grafana Data Sources

19) What is Loki?

Loki is a log aggregation system.

Developed by Grafana Labs.

Uses labels instead of indexing full logs.

Integrated with Grafana dashboards.

Correlates logs with metrics.

20) Basic Linux Commands

pwd – current directory

ls – list files

cat, less, head, tail – view files

uptime – system running time

cd – change directory

touch / mkdir – create files/folders

21) AWS Basics

AWS is a cloud computing platform.

EC2 – virtual servers.

S3 – object storage.

VPC – private network.

RDS – managed databases.

CloudWatch – monitoring.

Auto Scaling – automatic scaling.

ELB – load balancing.

22) AWS Session Manager

Secure EC2 access without SSH.

No key pairs required.

No inbound ports needed.

Audited and logged.

More secure than SSH.
