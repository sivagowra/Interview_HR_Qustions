Amazon EC2

EC2 is the main service used to run servers in AWS.

Amazon EC2 allows users to create virtual computers in the cloud. These virtual machines are called instances and can run Linux or Windows operating systems. EC2 removes the need to buy physical servers. Users can increase or decrease server size at any time. EC2 supports different instance types based on performance needs. It works closely with storage and networking services. Security is controlled using key pairs and firewall rules. EC2 is widely used to host applications and websites.

Elastic Load Balancer (ELB)

ELB helps manage traffic when many users access an application.

Elastic Load Balancer spreads incoming traffic across multiple servers. This prevents a single server from becoming overloaded. ELB improves application availability and reliability. It automatically sends traffic only to healthy servers. ELB supports HTTP, HTTPS, and TCP traffic. It works well with Auto Scaling to handle traffic changes. SSL security can be configured easily. ELB is important for high-traffic applications.

Auto Scaling

Auto Scaling keeps applications stable during traffic changes.

Auto Scaling automatically increases or decreases the number of EC2 instances. It helps applications handle high traffic without manual work. When traffic increases, new instances are added automatically. When traffic reduces, extra instances are removed to save cost. Auto Scaling uses rules based on CPU or memory usage. It improves fault tolerance by replacing unhealthy instances. It works with ELB for better performance. This service reduces downtime and cost.

Elastic Block Store (EBS)

EBS provides storage for EC2 instances.

Amazon EBS is a block storage service used with EC2. It works like a hard disk for virtual servers. Data stored in EBS is persistent even if the instance stops. Users can create backups using snapshots. EBS supports different volume types for speed and cost. It is reliable and secure. EBS is commonly used for databases. It ensures fast read and write operations.

Virtual Private Cloud (VPC)

VPC is the private network where AWS resources run.

Amazon VPC allows users to create an isolated network in AWS. It gives full control over IP addresses and subnets. Resources inside VPC can be public or private. Security groups and network ACLs control traffic. VPC allows secure communication between services. It supports VPN connections from on-premise networks. Flow logs help monitor traffic. VPC is the base of AWS infrastructure.

Amazon S3

S3 is used to store files and data safely.

Amazon S3 is an object storage service for storing data. It can store any amount of data with high durability. Files are stored in buckets and accessed using URLs. S3 is highly available and secure. It supports automatic lifecycle management. Encryption protects data at rest and in transit. S3 is used for backups and static websites. It integrates with many AWS services.

IAM

IAM controls who can access AWS resources.

IAM is used to manage users and permissions in AWS. It allows creating users, groups, and roles. Permissions are given using policies. IAM follows least privilege security principle. Multi-factor authentication increases security. Roles are used by services to access resources. IAM works across all AWS services. It is critical for cloud security.

Amazon Route 53

Route 53 connects domain names to applications.

Route 53 is a DNS service that converts domain names to IP addresses. It provides fast and reliable routing. Health checks help route traffic to healthy servers. It supports different routing policies. Route 53 integrates with ELB and CloudFront. Domain registration is also supported. It improves global application performance. Route 53 ensures high availability.

Amazon RDS

RDS makes database management easy.

Amazon RDS is a managed relational database service. It supports MySQL, PostgreSQL, Oracle, and more. AWS handles backups and patching automatically. Multi-AZ improves availability. Monitoring is done using CloudWatch. Encryption ensures data security. RDS reduces operational workload. It is used for application databases.

Amazon EFS

EFS is shared file storage for EC2.

Amazon EFS provides scalable file storage. Multiple EC2 instances can access the same data. Storage grows automatically when data increases. It is highly available and durable. EFS is mainly used for Linux workloads. It supports secure access. EFS simplifies file management. It is useful for shared applications.

SQS

SQS helps services communicate safely.

Amazon SQS is a message queue service. It stores messages until they are processed. It decouples application components. SQS improves scalability and reliability. Messages are processed asynchronously. FIFO queues maintain order. SQS integrates with Lambda and EC2. It is used in microservices.

SNS

SNS sends messages to many users.

Amazon SNS is a notification service. It sends messages to email, SMS, and applications. Messages are published to topics. Subscribers receive messages instantly. SNS supports event-driven systems. It integrates with AWS services. Access is controlled using IAM. SNS is used for alerts.

SES

SES is used to send emails from applications.

Amazon SES is an email service. It sends transactional and marketing emails. SES provides high delivery rates. It supports email authentication. It reduces email costs. Security features prevent spam. SES integrates with AWS services. It is used for application emails.

CloudWatch

CloudWatch monitors AWS resources.

CloudWatch collects logs and metrics. It helps monitor performance. Dashboards show real-time data. Alarms notify issues. It helps with troubleshooting. CloudWatch works with Auto Scaling. Logs help debugging. It is essential for monitoring.

CloudFront

CloudFront makes websites faster.

CloudFront is a CDN service. It delivers content from nearby locations. It reduces latency. Data is cached globally. It supports HTTPS security. CloudFront integrates with S3. It reduces server load. It improves user experience.

CloudFormation

CloudFormation automates infrastructure creation.

CloudFormation uses templates to create resources. It supports Infrastructure as Code. It reduces manual work. Resources are created consistently. Rollback handles failures. Templates are reusable. It supports automation. It is used in DevOps.

CloudTrail

CloudTrail tracks AWS activity.

CloudTrail records API calls. It logs user actions. Logs are stored in S3. It helps in audits. Security analysis is possible. It supports multi-region logging. CloudTrail improves visibility. It helps meet compliance needs.

ECS

ECS manages Docker containers.

Amazon ECS is a container service. It runs Docker containers easily. It integrates with AWS services. ECS supports EC2 and Fargate. It simplifies container management. IAM controls security. Monitoring is supported. ECS is used for microservices.

EKS

EKS runs Kubernetes on AWS.

Amazon EKS is a managed Kubernetes service. AWS manages the control plane. It integrates with AWS networking. It supports auto scaling. Worker nodes can be managed or serverless. EKS is highly available. It supports open-source tools. It is used for cloud-native apps.
