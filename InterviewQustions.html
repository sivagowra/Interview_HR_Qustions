ğŸ”¹ Process vs Thread

Feature Process Thread
Definition A process is an independent program in execution A thread is a small unit of execution inside a process
Memory Each process has separate memory space Threads share the same memory of the process
Communication Inter-process communication (IPC) is complex Thread communication is easy (shared memory)
Creation Time Process creation is slow Thread creation is fast
Context Switching Heavy (more overhead) Lightweight
Failure Impact If one process crashes, others are not affected If one thread crashes, the entire process may crash
Resource Usage Uses more system resources Uses less resources
Example Running Chrome and VS Code Multiple tabs in Chrome

3. Docker Server Down â€“ How to Troubleshoot

ğŸ³ Docker Server Down â€“ How to Troubleshoot

ğŸ”¹ Step 1: Check Docker service status

Look for:

  • active (running) âŒ or failed âŒ
  • Error messages in red

ğŸ”¹ Step 2: Check Docker service logs

Common errors:

  • Disk full
  • Permission denied
  • cgroup / storage driver issues

ğŸ”¹ Step 3: Check disk space (VERY COMMON ISSUE)

This will show exact startup error (useful for root cause).

ğŸ”¹ Step 5: Verify Docker configuration

Check for:

  • Wrong storage driver
  • Wrong log driver

After changes:

systemctl restart docker

ğŸ”¹ Step 6: Check kernel & cgroup compatibility

Look for:

  • cgroup driver mismatch
  • storage driver errors

ğŸ”¹ Step 7: Check OS & Docker version

Version mismatch can break Docker after OS updates.

ğŸ”¹ Step 8: Check running containers (if Docker is up)

docker ps -a

Check container logs:

docker logs <container_id>

ğŸ”¹ Step 9: Restart Docker & system (if needed)

systemctl restart docker
reboot

ğŸ”¹ Interview-Ready Answer (Short)

ğŸ”¹ What is Jenkins?

  • Notifications are sent if the build fails.

ğŸ”¹ How I Troubleshoot My Server

Server troubleshooting involves identifying, analyzing, and resolving issues affecting system performance, availability, or functionality.

ğŸ”¹ Step-by-Step Approach

  • Check System Health: CPU top, Memory free -h, Disk df -h, Processes ps -ef
  • Check Logs for Errors: System logs /var/log/messages, Application logs /var/log/nginx/error.log, Docker logs docker logs <container_id>, CloudWatch for AWS resources
  • Check Services Status: systemctl status nginx, systemctl restart docker
  • Identify Network Issues: ping <server-ip>, netstat -tulnp, ss -tulnp, Firewall iptables -L or ufw status
  • Disk & Storage Troubleshooting: du -sh /var/log/*, docker system prune -af
  • Application-Specific Checks: Docker docker ps -a, Kubernetes kubectl get pods -o wide, Database logs and connectivity
  • Restart Services or Server if Needed: systemctl restart <service>, reboot

ğŸ”¹ Real-Time DevOps Use Case

  • Observed high CPU via top
  • Checked Docker container logs: docker logs webapp
  • Identified memory leak in application
  • Restarted container: docker restart webapp
  • Cleared old logs: rm -rf /var/log/old_logs/*
  • Set CloudWatch alarms to monitor CPU and disk usage

ğŸ”¹ Interview-Ready Short Answer

I troubleshoot servers by monitoring system metrics (CPU, memory, disk), checking logs, verifying service and network status, identifying issues with applications or containers, and applying fixes such as restarting services, cleaning disk space, or redeploying containers. I also use tools like CloudWatch, Docker logs, and monitoring scripts for proactive issue detection.

ğŸ”¹ Why We Use grep Commands

grep is a command-line utility used in Linux/Unix to search for specific patterns or text in files or command output.

  • Quickly find specific lines, keywords, or errors in large log files
  • Can be combined with other commands (ps, docker logs, cat) to filter information
  • Supports regular expressions for advanced pattern matching
  • Used in monitoring, debugging, and automation tasks

ğŸ”¹ Common Use Cases

  • Search for a keyword in a file: 
    grep "error" /var/log/nginx/error.log
  • Case-insensitive search: 
    grep -i "failed" /var/log/syslog
  • Recursive search in directories: 
    grep -r "docker" /home/projects/
  • Search with line numbers: 
    grep -n "database" app.log
  • Combine with other commands: 
    ps -ef | grep nginx
docker logs webapp | grep "Exception"

ğŸ”¹ Real-Time DevOps Use Case

  • Filtered Docker container logs: docker logs webapp | grep "ERROR"
  • Checked running processes: ps -ef | grep java
  • Checked network connections: netstat -tulnp | grep 8080

ğŸ”¹ Interview-Ready Short Answer

We use grep to search and filter specific patterns or keywords in files or command output. It helps in debugging, log analysis, and monitoring by quickly identifying relevant information.

ğŸ”¹ Types of Load Balancers

Type Description Layer Use Case Example
Application Load Balancer (ALB) Routes traffic based on content (URL, headers, path) Layer 7 (HTTP/HTTPS) Microservices, HTTP apps, API routing AWS ALB, NGINX
Network Load Balancer (NLB) Routes traffic based on IP protocol data Layer 4 (TCP/UDP) High-performance, low-latency, TCP/UDP apps AWS NLB, HAProxy
Classic Load Balancer (CLB) Basic load balancer, supports HTTP, HTTPS, TCP Layer 4 & Layer 7 Legacy applications AWS ELB (Classic)
Global Load Balancer Distributes traffic across multiple regions Layer 7 Multi-region apps, disaster recovery AWS Global Accelerator, Cloudflare
Internal / Private Load Balancer Load balances traffic inside VPC/private network Layer 4 or 7 Internal microservices AWS ALB internal, NGINX
Hardware Load Balancer Physical device for traffic distribution Layer 4 & 7 Data centers, enterprise apps F5 Big-IP, Citrix ADC
Software Load Balancer Software-based, can run on VMs or containers Layer 4 & 7 Cloud-native apps HAProxy, NGINX, Traefik

ğŸ”¹ Simple Explanation

Load balancers distribute incoming network traffic across multiple servers to ensure high availability and reliability.

ALB â†’ Smart, content-based routing (HTTP/HTTPS).

NLB â†’ Fast, TCP/UDP-based, low-latency routing.

CLB â†’ Old-style basic balancing.

ğŸ”¹ Real-Time Example

  • Your web application has 3 EC2 instances running.
  • User requests come in; ALB distributes requests evenly based on HTTP path.
  • If one instance fails, ALB stops sending traffic to it.
  • NLB can be used for TCP-based apps like game servers or database connections.

ğŸ”¹ Interview-Ready Short Answer

Load balancers distribute incoming traffic across multiple servers to ensure high availability, scalability, and fault tolerance. Types include Application Load Balancer (Layer 7), Network Load Balancer (Layer 4), Classic Load Balancer, Global Load Balancer, Internal Load Balancer, and hardware/software load balancers.
