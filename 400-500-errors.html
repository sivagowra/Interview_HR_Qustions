HTTP Error Codes Documentation

Client Errors (4xx)

400 – Bad Request

Meaning: Server cannot understand the request.

Common Causes:

  • Invalid JSON
  • Missing required fields
  • Wrong request format
  • Incorrect headers

How to Resolve:

  • Validate form inputs in frontend
  • Send proper Content-Type (application/json)
  • Check API request body format
  • Add backend validation
401 – Unauthorized

Meaning: Authentication required.

Common Causes:

  • Missing JWT token
  • Expired session
  • Invalid API key

How to Resolve:

  • Send valid Authorization header
  • Refresh expired token
  • Implement login flow properly
402 – Payment Required

Meaning: Reserved for payment systems.

Common Causes:

  • Subscription expired
  • Payment failure
  • API usage limit requiring billing

How to Resolve:

  • Check payment/subscription status
  • Renew plan
  • Handle billing validation in backend
403 – Forbidden

Meaning: Authenticated but no permission.

Common Causes:

  • Role-based restriction
  • Accessing admin page without role
  • Blocked IP

How to Resolve:

  • Check user roles (RBAC)
  • Update permission policies
  • Verify backend access control logic
404 – Not Found

Meaning: Requested resource does not exist.

Common Causes:

  • Wrong URL
  • API endpoint typo
  • Deleted resource

How to Resolve:

  • Check route path
  • Verify backend routes
  • Handle 404 page properly in frontend
405 – Method Not Allowed

Common Causes:

  • Using POST instead of GET
  • Backend route only allows specific method

How to Resolve:

  • Check allowed HTTP methods
  • Update frontend request type
  • Modify backend route configuration
408 – Request Timeout

Common Causes:

  • Slow network
  • Large payload
  • Server overload

How to Resolve:

  • Increase timeout configuration
  • Optimize request payload
  • Improve backend performance
429 – Too Many Requests

Common Causes:

  • Too many API calls
  • Brute force login attempts
  • Rate limiting enabled

How to Resolve:

  • Implement request throttling
  • Add retry-after delay
  • Increase rate limit (backend config)

Server Errors (5xx)

500 – Internal Server Error
  • Code crash
  • Unhandled exception
  • Database failure

How to Resolve:

  • Add try-catch blocks
  • Check server logs
  • Validate inputs before processing
  • Debug backend code
501 – Not Implemented
  • API feature not developed
  • Method not implemented

How to Resolve:

  • Implement missing functionality
  • Update API documentation
  • Remove unsupported request
502 – Bad Gateway
  • Backend service down
  • Kubernetes pod crash
  • Reverse proxy issue

How to Resolve:

  • Restart backend service
  • Check container logs
  • Verify load balancer configuration
503 – Service Unavailable
  • High traffic
  • Server scaling issue
  • Maintenance mode

How to Resolve:

  • Enable Auto Scaling
  • Increase server resources
  • Implement load balancing
504 – Gateway Timeout
  • Slow database query
  • API taking too long
  • Network delay

How to Resolve:

  • Optimize queries
  • Increase timeout settings
  • Improve infrastructure performance
