What is Troubleshooting?

Troubleshooting is a step-by-step process used to identify, analyze, and fix problems in systems, software, hardware, or networks.


Step 1: Identify the Problem

Understand the issue clearly by asking the user about the problem, checking error messages, and observing symptoms.


Step 2: Gather Information

Collect details such as when the issue started, what actions were taken, and check logs or alerts related to the problem.


Step 3: Analyze the Problem

Analyze the collected information to find the possible root cause of the issue.


Step 4: Create a Possible Solution

Decide the best and safest solution to resolve the issue, starting with simple fixes.


Step 5: Implement the Solution

Apply the chosen solution carefully by following IT policies and standard procedures.


Step 6: Test the Solution

Verify whether the issue is resolved and confirm that the system is working properly.


Step 7: Document the Issue

Record the problem, root cause, and solution in the ticketing tool for future reference.


Step 8: Close or Escalate the Ticket

Close the ticket if the issue is resolved, or escalate it to higher-level support if needed.


Troubleshooting Process Flow

Identify → Gather Info → Analyze → Fix → Test → Document → Close/Escalate


Printer Not Connecting to System

When a printer does not connect to a system, it can be caused by cable issues, network problems, driver errors, or service issues. Follow the below troubleshooting steps to resolve the problem.


Step 1: Check Physical Connections

Ensure the printer is powered on and properly connected to the system using a USB cable or network cable. Check Wi-Fi connectivity for wireless printers.


Step 2: Check Printer Status

Go to Control Panel and open Devices and Printers. Check if the printer status shows Offline. Set the printer as default and change status to Online if required.


Step 3: Restart Printer and System

Turn off the printer and the system. Wait for a few seconds, then power on the printer first and the system next to clear temporary issues.


Step 4: Check Network Connection

For network printers, make sure the system and printer are connected to the same network. Verify the printer IP address and test network connectivity.


Step 5: Reinstall or Update Printer Driver

Remove the existing printer driver from Device Manager and install the latest driver from the printer manufacturer’s website.


Step 6: Restart Print Spooler Service

Open services.msc, locate Print Spooler service, restart it, and ensure the startup type is set to Automatic.


Step 7: Add Printer Again

Go to Devices and Printers, click Add Printer, and add the printer manually using the correct method or IP address.


Step 8: Test Printing

Print a test page to confirm the printer is working correctly and connected to the system.


Step 9: Escalate the Issue

If the issue is not resolved, collect error details and escalate the ticket to the L2 or hardware support team.

What is an IP Address?

An IP address (Internet Protocol address) is a unique number assigned to a device such as a computer, mobile, printer, or server. It helps devices identify each other and communicate on a network or the internet.


Why IP Address is Important

An IP address is used to identify devices, send and receive data, troubleshoot network issues, and enable internet and local network communication.


Types of IP Addresses

  • Private IP Address – Used within home or office networks (example: 192.168.x.x)
  • Public IP Address – Assigned by Internet Service Providers and used on the internet

How to Check IP Address in Windows

Press Windows + R, type cmd, press Enter, then type ipconfig and look for the IPv4 Address.


How to Check IP Address in Linux or macOS

Open the terminal and type ifconfig or ip a to view the IP address.


How to Check IP Address Using Settings

Open Settings, go to Network & Internet, select Wi-Fi or Ethernet, click on the connected network, and view the IP address details.


How to Check Public IP Address

Open a web browser and search for “What is my IP”. The public IP address will be displayed on the screen.

System Not Turning ON – Troubleshooting Steps

1. Check Power Supply

Ensure the power cable is properly connected.

Check if the power socket is working.

Try a different power cable or socket.


2. Check Power Button

Press the power button correctly.

Check if the power button is damaged.


3. Check Monitor

Make sure the monitor is turned ON.

Check HDMI or VGA cable connection.

Try another monitor if available.


4. Check Adapter or SMPS

Laptop: Check charging light.

Desktop: Check SMPS fan is running.

Try another adapter if possible.


5. Remove External Devices

Disconnect USB, printer, or external hard disk.

Try turning on the system again.


6. Check Internal Hardware

Remove and reinsert RAM.

Check motherboard cable connections.

Listen for beep sounds.


7. Check Battery (Laptop)

Remove the battery if possible.

Connect charger and power ON.


8. Final Step

If system still does not start, raise a support ticket.


Interview Answer

If a system is not turning on, I check power supply, cables, monitor, RAM, and hardware connections.

DHCP (Dynamic Host Configuration Protocol)

DHCP automatically gives IP addresses to devices in a network.

It saves time and avoids manual IP configuration.

Simple Meaning

When a device connects to a network, DHCP gives it an IP address automatically.

No need to set IP address manually.


Example

You connect your laptop to Wi-Fi.

DHCP assigns IP address, gateway, and DNS automatically.


Information Provided by DHCP

  • IP Address
  • Subnet Mask
  • Default Gateway
  • DNS Server

Why DHCP is Important

  • Saves time
  • Reduces IP conflicts
  • Easy network management

How to Check DHCP on Windows

Open Command Prompt

Type: ipconfig /all

Check "DHCP Enabled : Yes"


Common DHCP Issues

  • No internet connection
  • IP address shows 169.254.x.x
  • DHCP server not responding

Interview Answer

DHCP automatically assigns IP address and network details to devices.



DHCP Troubleshooting Steps

1. Check Physical Connection

Ensure network cable is connected properly or Wi-Fi is enabled.

Check router or switch power.


2. Check IP Address

Open Command Prompt.

Type: ipconfig

If IP starts with 169.254, DHCP is not working.


3. Renew IP Address

Open Command Prompt as Administrator.

Type: ipconfig /release

Then type: ipconfig /renew


4. Check DHCP Enabled

Type: ipconfig /all

Confirm "DHCP Enabled : Yes"


5. Restart Network Devices

Restart computer.

Restart router or DHCP server.


6. Disable and Enable Network Adapter

Go to Network Settings.

Disable and re-enable network adapter.


7. Check Firewall or Antivirus

Temporarily disable firewall.

Check if DHCP traffic is blocked.


8. Check DHCP Server

Verify DHCP service is running on server.

Check available IP address pool.


Interview Answer

If DHCP is not working, I check IP address, renew it, restart devices, and verify DHCP server.
