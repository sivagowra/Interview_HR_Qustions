CI/CD Pipeline Explanation

My CI/CD pipeline starts with code pushed to GitHub. Jenkins pulls the code, builds the Docker image, runs basic checks, and pushes the image to Docker Hub.

Then the application is deployed on EC2 or Kubernetes using Deployment and Service objects. This reduces manual deployment errors and improves release speed.

Kubernetes Deployment

First, I create a Docker image and push it to a container registry. Then I create a Kubernetes Deployment YAML to manage pods and replicas.

After that, I expose the application using a NodePort service so it can be accessed externally. I monitor the pods using kubectl and CloudWatch.
