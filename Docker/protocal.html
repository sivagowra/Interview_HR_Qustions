AWS Network Protocols – Full Explanation

What is a Protocol?

A protocol is a set of rules that defines how data is transmitted between systems over a network. In AWS, protocols control how traffic enters or exits resources like EC2, Load Balancers, RDS, and VPC.

1. TCP (Transmission Control Protocol)

TCP is a connection-oriented and reliable protocol. It ensures data is delivered accurately and in order.

Purpose: Used for applications where accuracy matters.

Port Service Usage
22SSHSecure Linux login
80HTTPWeb traffic
443HTTPSSecure web traffic
3306MySQLDatabase access
5432PostgreSQLDatabase access
3389RDPWindows remote login

AWS Usage: EC2, Load Balancer, RDS, ECS

2. UDP (User Datagram Protocol)

UDP is a connectionless and fast protocol. It does not guarantee delivery.

Purpose: Used when speed is more important than reliability.

Port Service Usage
53DNSName resolution
67/68DHCPIP assignment
123NTPTime synchronization
161SNMPMonitoring

AWS Usage: DNS, VPN, Monitoring tools

3. ICMP (Internet Control Message Protocol)

ICMP is used for network diagnostics. It helps check whether systems are reachable.

Type Purpose
Echo RequestPing
Echo ReplyPing response
Time ExceededTraceroute

AWS Usage: Troubleshooting connectivity issues

4. ICMPv6

ICMPv6 is ICMP for IPv6 networks. It handles routing and neighbor discovery.

AWS Usage: IPv6-enabled VPCs

5. HTTP & HTTPS

HTTP and HTTPS are application-layer protocols used for websites and APIs.

Protocol Port Purpose
HTTP80Normal web traffic
HTTPS443Secure encrypted traffic

AWS Usage: ALB, EC2, CloudFront

6. FTP / SFTP / FTPS

Protocol Port Purpose

7. Email Protocols

8. VPN & Security Protocols

9. ALL Traffic

Allows all protocols and ports. This is dangerous and should only be used temporarily.

Final Summary

TCP → Reliable applications
UDP → Fast services
ICMP → Network health
HTTPS → Secure web traffic
