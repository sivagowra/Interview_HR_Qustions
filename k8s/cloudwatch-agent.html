Amazon CloudWatch Agent Setup Guide (EC2 + Docker)

Step 1: Install CloudWatch Agent

# Download and install the agent
wget https://s3.amazonaws.com/amazoncloudwatch-agent/amazon_linux/amd64/latest/amazon-cloudwatch-agent.rpm
sudo rpm -U ./amazon-cloudwatch-agent.rpm

Step 2: Start CloudWatch Agent Configuration Wizard

sudo /opt/aws/amazon-cloudwatch-agent/bin/amazon-cloudwatch-agent-config-wizard

Step 3: Wizard Options

  1. Operating System: Linux (default 1)
  2. EC2 or On-Premises: EC2 (default 1)
  3. User to run agent: cwagent (default 1)
  4. StatsD daemon: No (default 2)
  5. CollectD metrics: No (default 2)
  6. Host metrics: Yes (default 1)
  7. Per CPU core metrics: No (default 2)
  8. EC2 dimensions: Yes (default 1)
  9. High resolution metrics: 60s (default 4)
  10. Default metrics config: Standard (default 2)

Step 4: Configure Logs

  1. Do you want to monitor log files? → Yes (1)
  2. System log: 
    Log file path: /var/log/messages
Log group name: messages
Log group class: STANDARD (1)
Log stream name: {instance_id} (default)
Retention: 7 days (choice 5)
  3. Do you want additional logs? → Yes (1)
  4. Docker container logs: 
    Log file path: /var/lib/docker/containers/*/*.log
Log group name: docker-logs
Log group class: STANDARD (1)
Log stream name: {instance_id} (default)
Retention: 7 days (choice 5)
  5. Do you want more logs? → No (2)

Step 5: X-Ray Traces

Do you want CloudWatch agent to retrieve X-Ray traces? → No (2)

Step 6: Store Configuration in SSM Parameter Store

Do you want to store config in SSM Parameter Store? → Yes (1)
Parameter name: AmazonCloudWatch-linux (default)
Region: your EC2 region (e.g., ap-south-1)
Credential: Use EC2 instance role / SDK (default 1)

Step 7: Start CloudWatch Agent

sudo systemctl start amazon-cloudwatch-agent
sudo systemctl enable amazon-cloudwatch-agent
sudo systemctl status amazon-cloudwatch-agent

Step 8: Verify in AWS Console

  • Metrics: CloudWatch → Metrics → CWAgent
  • Logs: CloudWatch → Log groups → messages, docker-logs
