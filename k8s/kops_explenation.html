Complete KOPS Kubernetes Cluster Setup – Detailed Explanation

Step 1: Install AWS CLI

Purpose: AWS CLI allows your EC2 instance to communicate with AWS services like EC2, S3, IAM.

curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"

This downloads the AWS CLI package.

unzip awscliv2.zip

This extracts the installation files.

sudo ./aws/install

This installs AWS CLI into your system.

aws --version

This verifies successful installation.

Why Important? Without AWS CLI, KOPS cannot create AWS infrastructure.

Step 2: Install KOPS

Purpose: KOPS (Kubernetes Operations) is used to create and manage Kubernetes clusters on AWS.

curl -Lo kops https://github.com/kubernetes/kops/releases/latest/download/kops-linux-amd64

Downloads the latest KOPS binary.

chmod +x kops

Makes the file executable.

sudo mv kops /usr/local/bin/kops

Moves KOPS into system path so you can run it globally.

kops version

Checks installed version.

What KOPS Does Internally:

  • Creates VPC
  • Creates EC2 instances
  • Creates IAM roles
  • Creates Security Groups
  • Configures Kubernetes components

Step 3: Install Kubectl

Purpose: Kubectl is the command-line tool to control Kubernetes cluster.

curl -LO https://dl.k8s.io/release/stable.txt

Downloads latest kubectl version.

chmod +x kubectl

Make it executable.

sudo mv kubectl /usr/local/bin

Move it to system path.

kubectl version

Verify installation.

Why Needed? After cluster creation, you use kubectl to:

  • Deploy applications
  • Check pods
  • Check nodes
  • Manage services

Step 4: Create S3 Bucket (State Store)

Purpose: KOPS stores cluster configuration inside S3 bucket.

aws s3 mb s3://kops.flm.k8s

This creates a new S3 bucket.

export KOPS_STATE_STORE=s3://kops.flm.k8s

This tells KOPS where to store cluster configuration.

Important: If S3 bucket is deleted, cluster management becomes difficult.

Step 5: Create Cluster Configuration

This command defines how your cluster should look.

kops create cluster \
--name kops.flm.k8s.local \
--zones us-east-1a,us-east-1b \
--master-size t2.medium \
--master-count 1 \
--node-size t2.micro \
--node-count 1 \
--master-volume-size 8 \
--node-volume-size 8

Explanation of Parameters:

  • --name: Cluster name
  • --zones: AWS availability zones
  • --master-size: EC2 type for master
  • --master-count: Number of master nodes
  • --node-size: Worker node instance type
  • --node-count: Number of worker nodes
  • --volume-size: Storage size in GB

Important: This step only creates configuration in S3, not AWS resources yet.

Step 6: Create AWS Infrastructure

kops update cluster --name kops.flm.k8s.local --yes --admin

What Happens Now?

  • EC2 instances are launched
  • VPC is created
  • Security groups are created
  • IAM roles are created
  • Kubernetes control plane starts

This is the step where real infrastructure is created.

Step 7: Verify Cluster

kubectl get nodes

This command checks whether nodes are ready.

If output shows:

  • STATUS: Ready → Cluster is working
  • STATUS: NotReady → Cluster still initializing

Step 8: Delete Cluster

kops delete cluster --name kops.flm.k8s.local --yes

This removes:

  • EC2 instances
  • VPC
  • Security groups
  • IAM roles

Always delete cluster if not using to avoid AWS billing.

Cluster Restart

If you deleted cluster and want to recreate:

export KOPS_STATE_STORE=s3://kops.flm.k8s

Then run create cluster and update cluster commands again.

Make sure S3 bucket still exists.

Simple Architecture Understanding

  • AWS CLI → Connects EC2 with AWS
  • KOPS → Creates Kubernetes infrastructure
  • S3 → Stores cluster configuration
  • Master Node → Controls the cluster
  • Worker Node → Runs applications (pods)
  • Kubectl → Manages the cluster
