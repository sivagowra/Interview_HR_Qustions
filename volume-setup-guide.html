EC2 Ubuntu — Volume Partition, Format & Mount Guide

1. Create Partition on /dev/xvdf

sudo fdisk /dev/xvdf

2. Format & Mount /dev/xvdf1

sudo mkfs.ext4 /dev/xvdf1
sudo mkdir /mnt/vol1
sudo mount /dev/xvdf1 /mnt/vol1

3. Check Mounted Volumes

df -h

✅ PART 1 — How to UNMOUNT & DETACH VOLUME (OLD INSTANCE)

1️⃣ Unmount the volume

If your volume is mounted at /mnt/vol1:

sudo umount /mnt/vol1

2️⃣ Confirm unmount

df -h

The volume should NOT appear in the output.

3️⃣ Detach from EC2

Go to AWS Console → EC2 → Volumes → Select Volume → Actions → Detach Volume

Wait until the state becomes available.

✔️ STEP 1 — Attach the Volume to the New EC2 Instance (RedHat)

In AWS Console → EC2 → Volumes:

  • Select your volume
  • Click Actions → Attach volume
  • Choose the new EC2 instance
  • Set device name: /dev/sdf

Linux will detect it as:

/dev/xvdf

✔️ STEP 2 — Check Attached Disks

Run this on the new EC2 instance:

lsblk

You should see output similar to:

xvdf    202:80   0   10G  0 disk
└─xvdf1 202:81   0   10G  0 part

Your actual data is inside xvdf1 (not xvdf).

✔️ STEP 3 — Create a Mount Point

Use the same folder:

sudo mkdir -p /mnt/vol1

✔️ STEP 4 — Mount the Partition (IMPORTANT)

Mount the partition:

sudo mount /dev/xvdf1 /mnt/vol1

⚠️ Do NOT mount /dev/xvdf — it contains no file system. Always use /dev/xvdf1 (the partition).

✔️ STEP 5 — Check Your Data

Run:

ls /mnt/vol1

6. Check Mounted Volumes

df -h

Now both volumes /mnt/vol1 and /mnt/vol2 are ready for storing data.

✔️ EBS Volume is Permanently Mounted

After adding the entry in /etc/fstab and running the commands:

sudo systemctl daemon-reload
sudo mount -a

Your EBS volume will mount automatically every time the instance starts. To verify, reboot the instance:

sudo reboot

After reboot, check:

df -h

If /mnt/vol1 is listed → Your EBS volume is permanently mounted.
