1Ô∏è‚É£ What is Terraform?

  • Terraform is an Infrastructure as Code (IaC) tool by HashiCorp.
  • Allows you to provision, manage, and version cloud infrastructure using code.
  • Supports AWS, Azure, GCP, Kubernetes, VMware, etc.
  • Declarative configuration: Define what you want, Terraform figures out how to create it.

2Ô∏è‚É£ Terraform Installation

Linux / Ubuntu

sudo apt update && sudo apt install -y wget unzip
wget https://releases.hashicorp.com/terraform/1.14.3/terraform_1.14.3_linux_amd64.zip

  • Extract and add to PATH
  • Run terraform -v

Mac (brew)

brew tap hashicorp/tap
brew install hashicorp/tap/terraform
terraform -v

3Ô∏è‚É£ Terraform Basic Workflow

Step 1: Create Configuration

provider "aws" {
  region = "us-east-1"
}

resource "aws_security_group" "sg" {
  name        = "terraform-sg"
  description = "Allow SSH traffic"

  ingress {
    from_port   = 22
    to_port     = 22
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]
  }

  egress {
    from_port   = 0
    to_port     = 0
    protocol    = "-1"
    cidr_blocks = ["0.0.0.0/0"]
  }
}

resource "aws_instance" "web" {
  ami                    = "ami-0532be01f26a3de55"
  instance_type          = "t2.micro"
  key_name               = "docker"
  vpc_security_group_ids = [aws_security_group.sg.id]

  root_block_device {
    volume_size = 8
  }

  tags = {
    Name = "Terraform-EC2"
  }
}

Step 2: Initialize Terraform

terraform init

Step 3: Validate Configuration

terraform validate

Step 4: Plan Infrastructure

terraform plan

Step 5: Apply Changes

terraform apply
# Or skip prompt
terraform apply -auto-approve

Step 6: Destroy Infrastructure

terraform destroy

  backend "s3" {
    bucket         = "my-terraform-state"

    dynamodb_table = "terraform-lock"
    encrypt        = true
  }
}

7Ô∏è‚É£ Terraform Features

Feature Description
Infrastructure as Code Define infra in code files
Immutable Infrastructure Changes create new resources instead of editing existing
Multi-Cloud Support AWS, Azure, GCP, etc.
State Management Tracks real infrastructure
Dependency Graph Creates resources in correct order
Modules Reusable templates for infra
Plan & Apply Preview changes before applying
Provisioners Run scripts on instances
Workspaces Manage multiple environments (dev/stage/prod)

8Ô∏è‚É£ Terraform Best Practices

  • Always use remote backend (S3 + DynamoDB)
  • Keep state files secure
  • Use modules for reusable infrastructure
  • Consistent naming (no hyphens in resource names)
  • Tag all resources
  • Always run terraform plan before apply
  • Backup tfstate before major changes

Question Sample Answer
