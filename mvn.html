Introduction to Apache Maven

Last Updated : 24 Jan, 2026

Apache Maven is the industry-standard build automation and project management tool for Java. While it is often seen as just a dependency manager, Maven is actually a complete project management framework. Unlike procedural tools like Ant or Make, Maven is declarative; you describe what your project is through the POM, and Maven automatically knows how to build it using predefined lifecycles.

Installing and Setting Up Maven

  1. Download Maven: Visit the official Maven website and download the latest version.
  2. Extract the Archive: Extract the downloaded archive to a suitable location.
  3. Set Environment Variables: Add the bin directory to the PATH variable.
  4. Verify Installation: Run mvn -version in terminal.

Core Concepts: The Maven Architecture

1. The POM (Project Object Model)

The pom.xml is the brain of a Maven project. It contains:

  • Identity: groupId, artifactId, version
  • Dependencies: External libraries like JUnit, Log4j
  • Plugins: Tools to compile, test, and package code

2. Repositories (Where the JARs Live)

Local Repository: Stores downloaded dependencies locally.

Example: C:\Users\asingh\.m2

Central Repository: Maintained by the Maven community.

Example: JUnit downloaded from repo.maven.apache.org

Remote Repository: Organization-specific repositories.

Example: http://nexus.company.com/repository/maven-releases

Key Terminologies

  • POM Files: XML files containing project configuration.
  • Dependencies and Repositories: External libraries stored in repositories.
  • Build Lifecycles: Ordered phases executed during build.
  • Build Profiles: Environment-specific configurations.
  • Build Plugins: Tools to perform specific tasks.

The Build Lifecycle: Phases and Goals

Default Lifecycle Phases

  • validate: Checks project configuration
  • compile: Compiles source code
  • test: Runs unit tests
  • package: Creates JAR/WAR
  • verify: Runs integration tests
  • install: Installs to local repository
  • deploy: Uploads to remote repository

Key Rule: Running a phase executes all previous phases automatically.

Dependency Management

Maven supports transitive dependency management. Adding one dependency automatically pulls all required dependent libraries.

Snapshot vs Release

  • Release: Stable and fixed version (e.g., 1.0.0)
  • Snapshot: Development version (e.g., 1.0.0-SNAPSHOT)

Build Profiles

Profiles allow different configurations for environments like Dev, Test, and Prod.

Example command: mvn package -P prod

Maven Directory Structure

  • src/main/java – Application code
  • src/main/resources – Configuration files
  • src/test/java – Unit tests
  • target – Build output

Use Cases of Maven

  • Java project builds
  • Dependency management
  • CI/CD integration
  • Automated testing
  • Project standardization and documentation

Maven in Test Automation

Maven helps manage test dependencies, execute tests automatically, and generate reports, ensuring consistent test execution.

Best Practices

  • Keep pom.xml clean
  • Use dependency management
  • Follow standard directory structure
  • Use profiles wisely

Essential Commands Cheat Sheet

Command Description
mvn clean Deletes the target folder
mvn compile Compiles source code
mvn test Runs unit tests
mvn package Creates JAR/WAR
mvn install Installs artifact to local repository
mvn dependency:tree Displays dependency hierarchy
